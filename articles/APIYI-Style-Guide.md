# APIYI 文章写作风格指南

## 核心写作原则

### 1. 语气特点
- 使用第一人称叙述，但要简洁自然："昨晚，我看到..."
- 直接对读者使用"你"，拉近距离感
- 保持专业性的同时带有对话感
- 表达要真实，不过度营销
- 用简单的感叹词表达情感："有点惊艳啊"

### 2. 内容结构
- 标题要直接点明主题，不怕争议（如："为什么不推荐用又旧又贵的 GPT-4？"）
- 开篇要有个人经验分享或观察
- 核心观点要加粗强调
- 分点论述要清晰，使用小标题和列表
- 结尾要有明确的行动建议

### 3. 表达特色
- 使用接地气的词汇："太强太卷了"
- 适当表达不确定性："还不知道在哪里试用"
- 避免过度承诺和夸张表述
- 删除未经确认的合作关系说法
- 保持内容的真实性和可信度

### 4. 格式规范
- 模型名称使用小写：`gpt-4`、`gpt-4o`
- 重要信息使用加粗：**关键提示**
- 版本号要精确：`GPT-4o-2024-11-20`
- 价格和数据要具体化

## 写作要点

### 1. 内容定位
- 面向技术初学者和开发者
- 强调实用性和成本效益
- 提供具体可行的建议
- 分享个人验和洞察

### 2. 论述方式
- 先抛出普遍现象
- 指出问题或误区
- 提供解决方案
- 给出具体建议

### 3. 语言风格
- 专业但不晦涩
- 直白但不粗糙
- 个性化但不随意
- 观点鲜明但有依据

### 4. 常用表达
- "我聊过很多用户..."（引入经验）
- "这里要注意..."（强调重点）
- "你可以..."（给出建议）
- "为什么这么说？"（引导思考）

## 示例段落

### 开篇示例
```markdown
我聊过很多用户，大家在测试 API 时，默认就预设：gpt-4。是的，GPT-4依然是许多用户的首选，这种现象背后有着多重原因...
```

### 观点表达示例
```markdown
然而，知名度高并不等同于性价比最优。我在这里旗帜鲜明的表达立场：**不推荐使用 GPT-4**。
```

### 建议给出示例
```markdown
最快的也是最稳的，你使用：gpt-4o，最省钱的你就用 gpt-4o-mini。
```

## 注意事项
1. 保持专业性的同时要让内容易于理解
2. 观点要有理有据，不是单纯的主观判
3. 给出的建议要具体且可操作
4. 适当使用类比和举例来解释复杂概念
5. 价格和性能数据要准确且及时更新
